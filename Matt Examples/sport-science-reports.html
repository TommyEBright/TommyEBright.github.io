CSV format reference
Required columns (any order, case-sensitive):
athlete_id
athlete_name
test_date
session_type
test_name
result
  • session_type must be pre or post (lowercase).
  • result must be numeric — no units.
  • Multiple trials = multiple rows for the same athlete + test + session.
  • Test names are free-form — use whatever names suit your battery. The tool computes means, SDs, CV%, change, CIs, Z/T-scores and TSA for any test.
  • Time-based tests are auto-detected for direction (sprint, agility, shuttle, _505, _time, modified_505). If yours doesn't match, name it consistently — e.g. Speed_30m, Sprint_Acceleration, Agility_T_Test, Modified_505.
  • Derived metrics (ASR, SRR, Modified 505 derived) need specific names: MAS (Maximal Aerobic Speed, m/s), Max_Sprint_Speed (m/s peak from 30m or 40m sprint), Speed_10m and Modified_505.
  • Optional metadata columns (free-form, any names): e.g. position, sex, group, age. Any extra columns beyond the required six become filter dropdowns in the Configure section, so you can focus the whole report on a sub-group.

Group analysis

Squad report

Group-level summary across athletes. Cohen's d, Bayes Factor (BF10) and group SWC live here because they apply to a sample of athletes, not individuals.

Test n Group change % change SWCgroup Cohen's d BF10 Squad CV% Verdict

SWCgroup = 0.2 × group SD (pre-session). In pre-post mode it's compared against the observed group change — |change| > SWC ⇒ meaningful. In baseline mode, SWCgroup is a forward-looking target: the threshold the future post-session change will need to exceed to count as meaningful at the squad level.

Squad change in SWC units — (Post − Pre) / SWC
Each axis = a test. Distance from centre = |(Post − Pre) / SWC|, direction-corrected. SWC = 0.2 × between-subject SD. Dashed rings mark 1× SWC (meaningful threshold), 2× SWC, and 3× SWC (substantial). Points are green for improvement, red for regression, grey if within SWC.

Athlete ranking

Total Score of Athleticism

A single holistic athleticism score per Turner (2014). TSA = sum of direction-corrected z-scores across the common test battery — sprint and agility tests are sign-flipped so a faster time produces a higher z. Athletes are ranked best to worst and RAG-rated by thirds.

Derived metrics

Automatically calculated from source tests where available.

Anaerobic Speed Reserve (ASR)

ASR = Max Sprint Speed (30m or 40m peak speed) − MAS

  • Unitsm/s
  • InterpretationReserve above aerobic ceiling

Sprint Reserve Ratio (SRR)

SRR = Max Sprint Speed ÷ MAS

  • SRR > 1.90Speed-biased (sprinters)
  • 1.80 – 1.90Hybrid
  • SRR < 1.70Endurance-biased (distance)

Speed_10m − Modified_505 — isolates change-of-direction ability

How it works

Everything runs in your browser — CSV never leaves your device.

Individual stats

  • Mean & SD — per session, from trials.
  • CV% = (SD ÷ Mean) × 100.
  • Change = post − pre; raw and as %.
  • 95% CI on the change — Welch standard error, two-tailed.
  • Individual SWC = 0.3 × within-subject SD. Change exceeds SWC ⇒ meaningful for that athlete.
  • Z-score (post) = (athlete post mean − squad post mean) ÷ squad post SD. T-score = 50 + 10·Z.
  • Z & T of change — how this athlete's change compares to the squad's change.

Squad / group stats

  • Group mean change — average across athletes.
  • Group SWC = 0.2 × group SD (pre).
  • Cohen's d = (group post mean − group pre mean) ÷ pooled group SD. Sample-level effect size.
  • Bayes Factor (BF10) — BIC approximation against the null of no change.
  • Squad CV% — average within-athlete CV across the squad.

Radar plots

  • Individual radar — T-scores at post-session for each test (50 = squad average, 60+ = above squad).
  • Squad radar — |group change| ÷ SWCgroup for each test. Beyond the 1.0 ring is meaningful.
Heads-up: The BF is a BIC approximation (Wagenmakers, 2007). It's a flag, not a verdict. For publication-quality Bayesian inference use R's BayesFactor package or Python's pingouin.
